# Data source
Power BI supports a wide range of data sources categorized into several groups. Understanding these categories and how they behave (import vs DirectQuery, gateway requirements, cloud/on-prem) is essential for the PL-300 exam.

### **1. Flat Files & Folders**

- Examples: Excel, CSV, JSON, XML, PDF, Folder.
- Mostly **Import mode**.
- Cloud files (OneDrive/SharePoint) → **No gateway**.
- On-prem files → **Gateway required**.
### **2. Databases**

- Examples: SQL Server, Oracle, MySQL, PostgreSQL, Snowflake, SAP, Databricks.
- Support **Import** + **DirectQuery** (depends on source).
- On-prem databases → **Gateway required**.
### **3. Power Platform**

- Examples: Dataverse, Power BI datasets, Dataflows, Analysis Services.
- Dataverse → Import + DirectQuery.
- Analysis Services → **Live connection only**.
- No gateway for cloud sources.
### **4. Azure**

- Examples: Azure SQL DB, Synapse, Data Lake, Blob Storage, Azure Cosmos DB.
- Cloud-based → **No gateway**.
- Supports Import/DirectQuery depending on source.
### **5. Online Services**

- Examples: SharePoint Online, OneDrive, Salesforce, Dynamics 365, Google Analytics.
- **Cloud-to-cloud** → No gateway.
- Mostly **Import mode**.
### **6. Others**

- Examples: Web URLs, OData, ODBC, R/Python scripts, APIs.
- ODBC/local scripts may require **gateway**.
- Often Import mode.
